# 폐쇄형 베타 데모 체크리스트

## 실행 준비

- 백엔드 실행 확인: `cd apps/api && python -m uvicorn app.main:app --reload`
- 프런트엔드 실행 확인: `cd apps/frontend && npm run dev`
- 데모 데이터 확인: `cd apps/api && python manage_db.py seed`
- 계정 확인: `student@unitflow.ai`, `student2@unitflow.ai`, `instructor@unitflow.ai`, `admin@unitflow.ai` / `password123`

## 1. 학생 메인 흐름

1. 학생으로 로그인한다.
2. 학생 홈에서 오늘의 핵심 할 일, 수시/정시 방향성, 목표대학 gap, 주간 전략 요약, low-confidence CTA를 확인한다.
3. 오늘의 핵심 할 일을 직접 추가/수정/삭제/완료 처리하고 저장한다.
4. AI 추천을 `replace`와 `append`로 각각 확인하고 source chip이 보이는지 확인한다.
5. 목표대학을 수정한 뒤 goal gap, student-visible strategy, 보완 CTA가 새로고침 없이 일관되게 바뀌는지 확인한다.
6. 진단 상세에서 취약 유형, 약한 과목/단원, evidence/signals, study recipe를 확인한다.
7. 전략 설계실에서 내 초안, 강사 승인본, AI 기본안, AI 보수안, 채팅, 검토 상태의 차이를 확인한다.
8. 전략 설계실에서 AI 추천을 받아 바뀐 필드가 화면에 표시되는지 확인하고 검토 요청을 보낸다.
9. 주간 플래너를 생성하고 일별 항목 완료/수행 시간을 기록한다.
10. 다시 만들기를 눌러 diff 요약이 1개 이상 표시되는지 확인한다.
11. 회고를 저장하고 추천 조정안 또는 재생성 권고가 표시되는지 확인한다.
12. 성장 리포트에서 점수, gap, 안정성, 실행률 변화를 확인한다.
13. What-if 시뮬레이터에서 수학 +3점, 주간 +4시간, 방향 가정을 바꿔 결과가 달라지는지 확인한다.
14. 공유 시험을 검색/생성하고 답안을 제출한 뒤 선택률/정답률/내 선택 비교를 확인한다.

## 2. 두 번째 학생으로 공유 시험 검증

1. `student2@unitflow.ai`로 로그인한다.
2. 같은 공유 시험에 답안을 제출한다.
3. 첫 번째 학생으로 돌아와 통계의 제출자 수, 선택률, 정답률이 바뀌었는지 확인한다.

## 3. 강사 보조 흐름

1. 강사로 로그인한다.
2. 대시보드에서 검토 필요한 학생과 추천 이유를 확인한다.
3. 시험/문항 CRUD와 결과 입력 또는 CSV 업로드를 확인한다.
4. 학생 재계산을 실행한다.
5. 학생 전략 검토 화면에서 AI 기본안, AI 보수안, 학생 초안, 승인본 diff를 확인한다.
6. 학생용 코칭 메시지와 내부 메모를 분리해 승인, 보류, 수정 요청 중 하나를 선택한다.

## 4. Admin 거버넌스

1. admin으로 로그인한다.
2. 대학 정책 CRUD를 확인한다.
3. 감사 로그, 변경 이력, 재계산 작업을 확인한다.
4. instructor 계정으로 admin-only 화면에 접근할 수 없는지 확인한다.

## 통과 기준

- 학생 핵심 화면이 placeholder/empty-state only 상태가 아니다.
- 오늘 핵심 할 일 저장/추천/새로고침 유지가 동작한다.
- 목표대학 변경 후 gap과 전략 상태가 일관되게 갱신된다.
- 전략 설계실에서 내 초안과 강사 승인본이 혼동되지 않는다.
- 플래너 다시 만들기와 회고 저장이 눈에 보이는 결과를 남긴다.
- What-if 시뮬레이터의 변동 배지는 실제 변경 필드와 맞다.
- 공유 시험은 2명 이상의 학생 제출 후 통계가 실제로 달라진다.
- 학생 데이터는 본인, 담당 강사, admin 범위를 벗어나 노출되지 않는다.
